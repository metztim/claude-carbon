# Contributing to Claude Carbon

Thanks for your interest in contributing!

## Quick Start

1. Fork this repo
2. Clone your fork locally
3. Open `ClaudeCarbon.xcodeproj` in Xcode
4. Make changes, test locally
5. Submit a Pull Request

## Using Claude Code

This project is vibe-coder friendly. You can explore and contribute using Claude Code:

```bash
claude --project /path/to/claude-carbon
```

Browse existing issues: `gh issue list`

Before starting work on a feature, comment on the issue to claim it.

## Ideas & Feature Requests

See [GitHub Issues](https://github.com/metztim/claude-carbon/issues) labeled `enhancement` for feature ideas.

To propose a new idea: Create an issue with `[Idea]` prefix.

## Areas for Contribution

### Methodology Improvements
- Better J/token estimates with research backing
- Regional carbon intensity options (currently US-only)
- More accurate PUE values

### UI/UX
- Gamification features (personal bests, achievements)
- New visualizations
- Accessibility improvements

### Documentation
- README clarity
- Methodology explanations
- Code comments

### Research
- AI energy consumption data
- Model comparisons
- Infrastructure efficiency data

## Code Style

- Swift, SwiftUI
- No external dependencies (keep it lightweight)
- Follow existing patterns in the codebase

## Requirements

- macOS 14.0+
- Xcode 15+
- Swift 5.9+

## Testing

Before submitting a PR:
1. Build and run in Xcode
2. Test with actual Claude Code usage
3. Verify menu bar icon and popover work correctly
